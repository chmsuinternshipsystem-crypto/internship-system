# Document Workflow Guide (Staff + Employer)

## Core Rule

Each required document has its own sequence.  
Not all documents follow one global passing process.

---

## Roles

- Student: submit student-origin documents and revisions
- Employer: validate company-origin requirements
- Instructor: primary intake/forwarding owner
- Chairperson: program compliance check
- Dean/VPAA: final institutional approvals for selected documents
- CIER/Legal: specialized review/signature steps

---

## Shared Status Lifecycle

`Missing -> Received -> Pending -> Returned for Revision -> Re-submitted -> Approved`

Optional terminal state:

`Rejected`

---

## Sequence Templates

- `MOA_LEGAL_FULL_CHAIN`:  
  `Instructor -> CIER -> Legal -> Employer -> Chairperson -> Dean -> VPAA`

- `PRE_DEPLOYMENT_CHAIN`:  
  `Instructor -> Chairperson -> Dean -> CIER`

- `MONITORING_CHAIN`:  
  `Employer -> Instructor -> Chairperson`

- `COMPLETION_CHAIN`:  
  `Employer -> Instructor -> Chairperson -> Dean -> CIER -> VPAA`

---
